# # # # # # # # # #
# Source Priority #
# # # # # # # # # #

Prefer sources in this order:

1. official documentation,
2. official repositories,
3. official package databases,
4. manpages and source code,
5. vendor support pages,
6. distribution wikis,
7. established technical databases,
8. issue trackers,
9. community reports,
10. forums and blogs only as low-confidence context.

# # # # # # # # #
# Evidence #
# # # # # # # # #

A factual entry must identify:

- what was found,
- where it was found,
- when it was checked,
- how trustworthy the source is,
- whether the item still needs review.

# # # # # # # # #
# Safety #
# # # # # # # # #

Do not research or store instructions for illegal bypass, credential theft, malware, account takeover, DRM circumvention or unauthorized access.
