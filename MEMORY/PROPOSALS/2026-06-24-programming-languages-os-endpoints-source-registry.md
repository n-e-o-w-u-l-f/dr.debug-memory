## Scope

This proposal seeds source records for broad programming-language discovery, standards bodies, query endpoints, package/ecosystem endpoints, and non-Linux operating-system discovery. It explicitly avoids claiming a complete list of all programming languages or all operating systems.

## Decision

The request for "all programming languages and endpoints" and "all other operating systems" cannot be fulfilled safely as a single canonical import. The correct GitHub knowledge addition is a discovery/source-registry layer that future scoped imports can use.

## Evidence model

- Official standards bodies/project docs: E3 SOURCE_CONFIRMED candidates.
- Knowledge graphs and community catalogues: E2 DISCOVERY_ONLY until confirmed by upstream.
- Package indexes and ecosystem registries are volatile and should be queried at use time.
