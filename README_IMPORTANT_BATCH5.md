# Dr.Debug-MEMORY Research Batch 5

Date: 2026-06-18
Mode: PUBLIC_SAFE_PROPOSAL_ONLY
Purpose: Continue technical research and extend the Batch 4 proposal with stronger networking, device-identifier, firmware/build and repair-source structures.

This package is not a direct GitHub commit and does not grant Dr.Debug public repository write access. It is a reviewable technical import proposal for the private Dr.Debug-MEMORY workflow.

## Added in Batch 5

- RFC/IANA-based network protocol source registry.
- Network foundations for IPv4 private ranges, CGNAT, NAT behavior, IPv6, NDP, DHCPv6, mDNS/DNS-SD, DoH/DoT, TCP, HTTP, SMB and SSH.
- Network diagnostic matrix with safe read-only commands for Linux, Windows and macOS.
- Network error signatures for routing, TCP, SSH, SMB, DNS, HTTP and FTP passive-mode failures.
- Device identifier authority records for PCI IDs, USB IDs, systemd-hwdb, LVFS/fwupd and Windows PnP IDs.
- Android build/recovery source rules for AOSP, LineageOS, TWRP, Samsung Open Source, SamMobile, SamFW and XDA-adjacent terminology.
- Samsung firmware/build taxonomy with BL/AP/CP/CSC/HOME_CSC/PIT fields and source-strength rules.
- Repair source policy for iFixit, ManualsLib, IPC-Computer, GeniusParts and Archive.org.
- Retro hardware source seeds, including 3dfx.
- Additional firmware terminology notes to separate firmware, NAND/bootloader images, ROMs, recoveries and modding scene terms.
- Second-pass evaluation of user-requested websites.
